Data Localization Suite (DLS)

An enterprise add-on that lets customers choose where Cloudflare inspects and stores data — without sacrificing security or performance.

1

Geo Key Manager (GKM)

Restricts which data centers hold your private SSL/TLS keys. Data centers without the key request a Session Key from an authorized one — adding minimal latency to the initial handshake only.

GKM v2 (Closed Beta):
Supports custom country allowlists/blocklists beyond the original US/EU presets.

2

Regional Services (RS)

Traffic is globally ingested with L3/L4 DDoS protection everywhere, but TLS termination and HTTP processing (Bot Management, WAF, Cache, Workers, Load Balancing) only happen inside the configured region.

DNS-level setting:
A US user connecting to an EU-regionalized hostname is routed to an EU data center for processing.
Learn more
3

Customer Metadata Boundary (CMB)

An account-level setting ensuring Customer Logs (traffic metadata with your Account ID) are stored exclusively in the EU or US. Combine with Logpush to export logs to your own storage.

Account-wide:
CMB applies to all services on the account.

DLS Components Stack

Customer Metadata Boundary (CMB) Account-level — Customer Logs (as defined by Privacy Policy)
Geo Key Manager (GKM) Certificate-level — Private key storage region (can cover multiple hostnames)
Regional Services (RS) DNS-level — TLS termination & HTTP processing region (per hostname)
Complementary (Add-On): Keyless SSL
For organizations that cannot allow private keys to leave their infrastructure, Keyless SSL keeps the private key on the customer’s own Hardware Security Module (HSM) or key server. During TLS handshakes, Cloudflare sends only the specific cryptographic operation to the customer’s key server over a mutually-authenticated TLS tunnel or Cloudflare Tunnel — the private key never touches Cloudflare’s network.
