Jurisdictional Restrictions

Built-in, product-level features to restrict data storage to specific geographic regions at creation time.

Supported Jurisdictions

Restricts data storage to data centers within the European Union. Helps comply with GDPR and EU data residency requirements.

Restricts data storage to FedRAMP-compliant data centers within the United States. Required for US government workloads.

Products with Jurisdiction Support

R2 Object Storage

Set jurisdiction on bucket creation via the dashboard or Wrangler CLI. Once set, all objects in the bucket are stored exclusively in the selected region.

D1 SQLite Database

Set jurisdiction at database creation. The primary database and all replicas are stored exclusively within the selected region.

Durable Objects

Restrict DO namespaces to a jurisdiction using the jurisdiction() API. All persisted data for the Durable Object stays within the selected region.

Important:
Jurisdictions can only be set at creation time and cannot be changed later. Workers may still access jurisdiction-restricted resources from anywhere — the restriction controls where data itself is stored (data-at-rest).
